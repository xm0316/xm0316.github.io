Git commands

Git 的一些命令操作

Git 的操作

安装

安装设置

  • git config –global user.name/email “…”

与 GitHub 关联

  • 生成 ssh key

    • ssh-key -t rsa -C “…email”

  • repository 关联

    • Remote：create a new repo

    • Local：关联

      • http 协议 or ssh 协议

        • https 协议每次 push 需要输入密码

      • associate

        • git remote add origin git@github.com：xm0316/Git_command.git
          git branch -M main
          git push -u origin main # origin：远程仓库名，main：本地仓库主分支名

    • 删除关联关系

      • git remote -v 查看远程库信息
      • git remote rm origin(远程分支名称)

文件操作

创建版本库 repo

  • git init

本地文件提交

  • 添加文件到暂存区

    • git add “…”

      • git add –all

  • 提交文件到版本库(本地仓库)

    • git commit -m “…提交说明”

      • git commit -am “…提交说明”，直接提交到版本库

文件比较 diff

  • git status

    • 掌握仓库当前的状态/变化情况

  • git diff

    • 查看文件在工作目录与暂存区的差别。
      如果还没 add 进暂存区，则查看文件自身修改前后的差别。

      • git diff –stat： 显示变化摘要而非整个 diff

    • git diff –staged/cached

      • 查看暂存区与版本库的区别

    • git diff

      • 查看工作目录同 Git 仓库指定 commit 的内容的差异。
      • =HEAD 时：查看工作目录同最近一次 commit 的内容的差异。

查看提交日志

  • git log

    • 显示从最近到最远的全部日志

  • git log –pretty=oneline

    • 单行显示日志(还有其他的参数)

  • git reflog

    • 显示可引用的历史版本记录

      • 按提交的命令显示，可以查看任意的提交点

版本回退 reset

  • 在 Git 中，用 HEAD 代表当前版本，上一个版本就是HEAD^，再上一个版本就是HEAD^^，依次类推,更多的可以使用HEAD~100

  • git reset –hard

    • 回到指定的已提交版本，丢弃版本库的变化

管理修改

  • git 跟踪并管理的是修改(暂存区的文件)

  • git diff HEAD –

    • 查看工作区和版本库最新版本的区别，指定版本见之前的命令

撤销修改

  • git status：检查文件变化/修改情况

  • 丢弃工作区的修改

    • git check –

      • git restore

  • 丢弃暂存区的修改

    • git reset HEAD

      • git restore –stage

  • 丢弃版本库的修改

    • git reset –hard

      • --- HEAD^

删除文件

  • 从版本库中删除一个文件

    • git rm
    • 并且提交：git commit -m “…”

  • 从版本库恢复文件

    • git checkout –
    • 用版本库的版本替换工作区的版本

远程仓库

添加远程仓库

  • GitHub 上的仓库既可以作为备份，又可以让其他人通过该仓库来协作

  • 关联远程仓库

    • git remote add origin git@github.com：账户名/仓库名.git

  • 首次推送

    • git branch -M main (给本地仓库改名)
      git push -u origin master
    • git push origin master

  • SSH 警告

    • 首次使用 Git 的 clone 或 push 命令连接 github 时，会提示你验证 RSA key 信息

删除远程仓库

  • 查看远程库信息

    • git remote -v

  • 解除远程库和本地库的绑定关系

    • git remote rm origin
